🤖 Agent System Architecture Overview

An internal reference page detailing the role coordination and operational flow of the OpenClaw agent setup.

🧠 The Core Coordination Layer

The system operates around a **Main Coordinator** (the primary agent/interface). This agent is responsible for context switching, task decomposition, and tool selection based on the user's input and overall system state. It reads the "SOUL" and "USER" context files to maintain persona and user needs.

*The diagram illustrates data flow: Input $\\rightarrow$ Coordinator $\\rightarrow$ (Context/Tools) $\\rightarrow$ Specialist Agents $\\rightarrow$ Output/Status Updates. Background tasks are handled by the Heartbeat loop.

🧩 Tool Orchestration Flow

The system uses specialized tools, managed by the coordinator, to break down complex requests. Key workflows include:

  • Information Retrieval: Utilizing `memory_search` or `ollama_web_search` to gather external or historical data.
  • File System Management: Employing `read`, `write`, or `edit` to manipulate project source code or local notes (e.g., in `TOOLS.md`).
  • Output Generation: Using `message` or `message` tools to deliver final content or updates to the appropriate channel.

🕰️ Automation & Continuity

The process is enhanced by background maintenance tasks:

  • Memory Maintenance: Periodically reviewing `memory/YYYY-MM-DD.md` files and consolidating key learnings into the long-term MEMORY.md to improve future performance.
  • Heartbeats: Proactive checks for urgent emails or upcoming calendar events, ensuring the user is always aware of critical information without being intrusive.
